✦ Input Guide

How to structure your input file

Use these formatting rules in your .docx file so our tool can parse it perfectly.

Title Style

Paper Title

Your paper title should be the very first paragraph in the document using the Title Word style (or plain text as the first line).

Title (first line) → Deep Learning Approaches for Fault Detection in Power Grid Systems
Heading 1 — Sections

Section Headings

Use Word's Heading 1 style for all major IEEE sections like Introduction, Methodology, Results, Conclusion, etc.

Heading 1 → Introduction Related Work Methodology Results
Heading 2 — Subsections

Subsection Headings

Use Word's Heading 2 style for subsections within a section. These will be auto-labelled A, B, C… in IEEE style.

Heading 2 → Dataset Overview Model Architecture Evaluation Metrics
Normal — Body Text

Paragraph / Body

Keep body text as Normal style in Word (12pt Times New Roman recommended). Avoid custom styles for regular paragraphs.

Normal style, 12pt The proposed method achieves a 94.3% classification accuracy across 5 benchmark datasets…
References

References Section

Add a Heading 1 labeled "References" and list them in IEEE citation format — numbered, bracketed. Add references manually after download.

Heading 1: References [1] A. Smith, "Title of paper," in Proc. IEEE Conf., 2023, pp. 1–5. [2] B. Jones et al., "Another paper," IEEE Trans., vol. 12…
Abstract & Keywords

Abstract & Keywords

Write the word "Abstract" as a plain paragraph (Normal style), then the abstract body below it. Similarly, write "Keywords" followed by the keyword list.

Normal: Abstract ↓ body text follows Normal: Keywords ↓ keyword list follows
💡 Pro Tips

Things to double-check before uploading

Style order: Title → Abstract → Keywords → Heading 1 sections.  •  Authors: Fill in the author form above — they'll be embedded in the output.  •  Tables: Caption goes above the table using "Table X: Caption" format.  •  References: Add them manually in the downloaded .docx (current limitation).  •  Figures: Not yet supported — add images manually after conversion.  •  Avoid: Text boxes, custom fonts, multi-column manual layouts.
